23. Razonar sobre la falsedad o no de las siguientes a?rmaciones: 
a) 'c' es una expresión de caracteres.

Verdadero. Al introducir una letra entre comillas simples, el ordenador lo almacena e interpreta como
un número entero sencillo (con un máximo de tres dígitos). En este caso se almacenará el valor 99 (el valor 
correspondiente a la letra c en la tabla ASCII). En definitiva dicho tipo de dato recibe el nombre de carácter.

b) 4<3 es una expresión numérica.

Falso. Aunque a simple vista pueda parecer una expresión numérica, esta se trata en realidad de una
expresión lógica, puesto que aparece en ella un "token" o símbolo propio de las variables booleanas 
o lógicas. Se trata del simbolo "<".

c) (4+3)<5 es una expresión numérica. 

Falso. A pesar de que aparezca el símbolo +, sigue tratandose de una expresión lógica, puesto que sigue 
presentando el token de <, es decir para que dicha función operase correctamente, debería estar asignada
a una variable de tipo bool ( que se corresponde con las expresiones lógicas).

d) cout << a; da como salida la escritura en pantalla de una a.

Falso. Para que ocurriese lo que se postula en el enunciado, la letra a debería de aparecer entre comillas
dobles ("a" de está forma si se imprimiría) Sin embargo, tal y como se redacta en el enunciado, la consola 
mostraría en pantalla el valor que tenga almacenado la variable a, previamente asignada.
 
e) ¿Qué realiza cin >> cte, siendo cte una constante entera? 

El comando que muestra el enunciado, sería el cin de la siguiente variable.
	const int cte;
Una vez dicho esto, debemos destacar que tenemos dos posibilidades:
- Si la variable cte no tiene ningún valor asignado, al realizar el cin, la variable cte almacenará 
el valor tecleado por el usuario. Una vez que este haya sido introducido, no se podrá modificar 
posteriormente.
- Si la variable cte ya tenía un valor asignado, el cin no modificará nada, por mucho que lo desee
el usuario, ya que esta es la principal función que cumple el comando "const", nos permite almacenar
un valor de forma que no pueda ser modificado posteriormente.
